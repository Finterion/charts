# Changesets

Hi! This folder has been automatically generated by `@changesets/cli`, a tool that helps you and your team to manage versioning and changelogs with a focus on monorepos.

## Workflow

1. When you make a change worth releasing, run:

   ```bash
   pnpm changeset
   ```

   It will ask you which packages changed and whether the change is a `patch`, `minor`, or `major` bump, then prompt for a short summary. The result is a markdown file in this folder.

2. Commit the changeset together with your code change.

3. The release workflow (`.github/workflows/release.yml`) will pick up changesets on `main`, open a "Version Packages" PR that bumps versions and updates `CHANGELOG.md`, and on merge will publish to npm.

For more, see <https://github.com/changesets/changesets/blob/main/docs/intro-to-using-changesets.md>.
